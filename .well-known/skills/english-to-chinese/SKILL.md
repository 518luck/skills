# 英文到中文翻译

将用户提供的英文内容翻译为自然、准确、符合中文表达习惯的简体中文。

## 使用场景

- 当用户要求把英文文本翻译成中文时使用。
- 当用户说“翻译成中文”“英译中”“中文版本”“帮我翻译下面内容”等类似请求时使用。
- 当用户提供英文文档、邮件、说明、报错、评论、提交信息或技术内容并要求中文表达时使用。
- 不要用于中文翻译成英文。
- 不要用于总结、改写、润色、扩写或解释英文内容，除非用户明确要求。
- 不要把翻译任务扩展成本地化策略、术语表维护或写作建议，除非用户明确要求。

## 工作流程

1. 确认用户要翻译的英文范围。
2. 识别文本类型、语气、受众和格式约束。
3. 翻译为简体中文，保留原文含义、语气和信息顺序。
4. 保留原有 Markdown、列表、标题、引用、链接、表格和代码块结构。
5. 检查技术术语、专有名词、变量名、命令、路径和 API 名称是否被错误翻译。
6. 只输出翻译结果；除非用户要求，不添加解释、总结或额外说明。

## 翻译规则

- 使用自然、准确、符合中文阅读习惯的简体中文。
- 忠实保留原文含义，不遗漏、不增添、不擅自纠错。
- 保留原文语气，例如正式、礼貌、警告、轻松、营销或技术说明。
- 技术术语优先使用常见中文译法；没有稳定译法时保留英文。
- 产品名、品牌名、库名、框架名、协议名、命令、参数、变量名、函数名、类名、文件路径、URL、邮箱地址不要翻译。
- 代码块内容默认不翻译；代码块外的说明文字需要翻译。
- Markdown 链接的展示文本可翻译，URL 不要修改。
- 保留数字、版本号、错误码、单位和占位符。
- 遇到双关、缩写或上下文不足的内容，优先给出最可能的自然译法，不要长篇解释。

## 输出格式

```markdown
[直接输出中文翻译，保留原文结构]
```

如果用户要求对照格式，使用：

```markdown
| English | 中文 |
| ------- | ---- |
| 原文    | 译文 |
```

## 验证

- 检查是否所有英文主体内容都已翻译。
- 检查是否保留 Markdown 结构、代码块、链接、路径、命令和变量名。
- 检查译文是否自然、准确，没有机械直译。
- 检查是否没有添加用户未要求的解释、总结或建议。
- 如果发现术语、格式或含义错误，修正后重新检查。

## 注意事项

- 用户只要求翻译时，最终回答不要加“以下是翻译”等前缀。
- 不要因为英文原文有语法问题就擅自改写含义。
- 技术文档中的 `must`、`should`、`may` 要准确体现强制程度。
- 错误信息和日志中的关键英文标识、错误码、字段名通常应保留英文。
