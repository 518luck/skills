# Skills 设计文档

本目录维护所有 Skill 的设计文档。每个子目录对应一个 Skill 或一组相关 Skill 的设计方案，记录背景分析、职责划分、架构决策和演进路线。

## 目录约定

- 每个子目录对应一个 Skill 或一组关联 Skill。
- 子目录内至少包含一个设计文档（本文档），记录"为什么这样设计"和"将来怎么演进"。
- 设计文档是给人读的，不是给 AI 读的；Skill 本身的执行指令仍在 `.well-known/skills/` 中。
