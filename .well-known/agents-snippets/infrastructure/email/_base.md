# 邮件发送通用规则

## 发送流程

- 邮件发送必须通过适配层调用，禁止在业务代码中直接初始化邮件 SDK。
- 适配层负责 provider 连接、模板渲染、错误处理和降级策略。

## 模板

- 邮件模板与前端组件体系分离；不要在邮件模板中使用前端 UI 组件库。
- 模板中的样式必须内联；邮件客户端不支持外部 CSS。
- 模板必须做响应式适配；确保在移动端邮件客户端中可读。

## 错误处理

- 邮件发送失败不应阻塞主业务流程；使用异步发送或队列。
- 记录发送失败的邮件，提供重试机制。
- 缺少邮件服务 API Key 时，跳过发送并返回空结果；调用方可能依赖这种降级行为，修改前必须检查。

## 禁止事项

- 禁止在邮件内容中包含敏感信息（密码、token 的完整值）。
- 禁止在业务代码中硬编码邮件地址；使用配置或常量。
- 禁止同步发送邮件阻塞用户请求。

## 变量

无额外变量。适配层路径在语言+包文件中通过 `{infrastructure_dir}` 指定。
