# 子目录 AGENTS.md 规则

本文件只处理子目录 `AGENTS.md` 的生成或重写。

## 目标

子目录 `AGENTS.md` 是局部规则文件。默认只生成一个简洁骨架，让用户后续自行补充；不要因为扫描到大量源码就自动写成冗长规范。

## 父链读取

生成前必须按顺序读取规则链路：

1. 确认仓库根目录和目标目录。
2. 从仓库根目录到目标目录父级，逐级查找已存在的 `AGENTS.md`。
3. 按从上到下的顺序读取所有祖先 `AGENTS.md`。
4. 如果目标目录已有 `AGENTS.md`，也要读取并作为更新基础。
5. 读取目标目录内的 `README*`、局部配置、测试文件、入口文件、包配置和邻近源码。

示例：目标是 `packages/foo/AGENTS.md` 时，需要检查：

```text
AGENTS.md
packages/AGENTS.md
packages/foo/AGENTS.md
```

不存在的父级文件不用创建，除非用户明确要求。

## 继承与去重

先建立规则分类表，再写文件：

- `继承`：父级已有且仍适用于本目录的规则。
- `局部补充`：只适用于目标目录的新规则。
- `局部覆盖`：目标目录需要替换父级规则的内容。
- `冲突待确认`：项目证据不足，无法判断是否覆盖父级。

子目录文件默认只写最少必要的 `局部补充` 和 `局部覆盖`。不要重复 `继承` 规则，也不要把父级规则总结成单独的“继承说明”章节。

常见去重规则：

- 不重复父级已有的 AI 行为指令。
- 不重复父级已有的提交、PR、安全或测试总原则。
- 不重复父级已有的全局安装、lint、类型检查、构建命令。
- 不生成局部命令章节，除非用户明确要求。
- 不生成局部测试规则章节，除非用户明确要求。
- 如果父级规则已经过时，不要在子目录里隐式修正；提醒用户应更新父级文件。

## 内容边界

子目录应该包含：

- 适用范围。
- 目标目录结构的最小骨架。
- 用户需要后续补充的局部代码规则章节。
- 本目录容易出错的事实或边界。

子目录不应该包含：

- 根目录项目概览、全局 AI 行为、全局提交规范等重复内容。
- 与目标目录无关的兄弟目录规则。
- 没有证据的局部命令或架构限制。
- 继承说明章节。
- 局部命令章节，除非用户明确要求。
- 局部测试规则章节，除非用户明确要求。
- 从源码扫描结果自动扩写出的大量细节。

## 推荐结构

默认生成以下最小骨架。除非用户明确要求，不要增加“继承说明”“局部命令”“局部测试规则”等章节。

```markdown
# <目录名> — AI Agent 局部指南

> 本文件继承上级 `AGENTS.md`，仅记录本目录的额外规则和易错边界。

## 适用范围

- 适用于 `<target-path>/**`。
- 上级 `AGENTS.md` 仍然有效，除非本文件明确覆盖。

## 本目录结构

- `<path>/`：<用途>
- `<path>/`：<用途>

## 局部代码规则

- <只写本目录必须额外遵守的规则。>

## 注意事项

- <本目录容易出错的事实。>
```

## 可选章节

只有用户明确要求时，才添加以下章节：

- `## 覆盖父级规则`：目标目录确实要覆盖父级规则时添加。
- `## 局部命令`：用户明确要求记录本目录命令时添加。
- `## 局部测试规则`：用户明确要求记录本目录测试规则时添加。
- 其他项目特定章节：用户明确要求或已有文件已经包含时才维护。

## 生成强度

- 默认只生成骨架，不主动填充所有扫描到的目录、命令和规则。
- 如果用户说“我自己补充”“只要架子”“先给模板”，只生成推荐结构中的章节和最少必要说明。
- 如果用户要求“帮我补全规则”，再根据父链和目标目录证据补充少量关键规则。
- 补全规则时也要保持克制，每个章节只写会影响 AI 行为的局部约束。

## 验证清单

- 已读取目标目录链路上的所有祖先 `AGENTS.md`。
- 文件明确写出适用范围。
- 文件声明继承上级规则。
- 没有重复父级、爷爷级或更高层的通用规则。
- 没有默认生成“继承说明”“局部命令”或“局部测试规则”章节。
- 没有从源码扫描结果自动扩写出冗长细节。
- 如果用户要求只生成骨架，允许保留少量待补充占位；否则不要保留 `<占位符>`、`[待补充]`、`TODO`。
