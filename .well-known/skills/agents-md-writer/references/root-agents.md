# 根目录 AGENTS.md 规则

本文件只处理仓库根目录 `AGENTS.md` 的生成或重写。

## 目标

根目录 `AGENTS.md` 是项目级 AI 工作指南，应该提供所有目录都适用的全局事实、全局命令、全局约束和子目录规则导航。

## 上下文收集

生成前必须读取能证明项目约定的文件，优先级如下：

1. 已存在的根目录 `AGENTS.md`。
2. `README*`、`CONTRIBUTING*`、`docs/` 中的开发说明。
3. 包管理和运行时配置：`package.json`、锁文件、`pyproject.toml`、`Cargo.toml`、`go.mod`、`Makefile`、`justfile` 等。
4. 质量工具配置：测试配置、lint 配置、formatter 配置、TypeScript 配置、CI 配置。
5. PR 模板、提交规范、发布说明、安全或合规文档。
6. 顶层目录结构和已有子目录 `AGENTS.md`。

不要只凭目录名猜测技术栈或命令。找不到证据时省略该项，或在最终说明中写明无法确认。

## 内容边界

根目录应该包含：

- 项目是什么、主要技术栈是什么、核心边界是什么。
- 默认分支、包管理器、运行时版本、虚拟环境等全局元信息。
- 全局安装、开发、测试、lint、类型检查、构建命令。
- 所有目录都适用的代码风格和 AI 行为规则。
- 全局安全、合规、提交、PR、CI 或发布规则。
- 已存在或推荐创建的子目录 `AGENTS.md` 导航。

根目录不应该包含：

- 只适用于单个模块、单个测试目录或单个客户端的细节。
- 大段背景文档、API 说明或通用教程。
- 无法从项目文件或用户明确说明推导出的命令。
- 与子目录已有 `AGENTS.md` 重复的局部规则。

## 生成规则

- 优先写短列表、表格和可复制命令。
- 每条命令都写成完整命令；如果必须在特定目录运行，写明目录。
- 如果全量测试很慢，写清如何运行相关测试，不要要求默认裸跑全量测试。
- 如果项目已有子目录 `AGENTS.md`，根目录只做导航，不复制其中细节。
- 如果项目适合拆分子目录规则，在根目录加“子目录 AGENTS.md”导航，但不要替不存在的文件编造具体规则。

## 验证清单

- 文件名是根目录 `AGENTS.md`。
- 没有 YAML frontmatter。
- 没有 `<占位符>`、`[待补充]`、`TODO`。
- 每个命令都有项目文件依据。
- 章节只包含全局规则，没有混入局部模块细节。
- 如果引用子目录 `AGENTS.md`，路径真实存在或明确标注为建议创建。
