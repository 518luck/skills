# Git 快速提交

检查变更、生成中文约定式提交信息、暂存并提交所有本地变更。不推送。

## 工作流程

1. 运行 `git status --short` 和 `git diff --stat` 快速了解变更范围。
2. 检查是否有 `node_modules`、`.env`、`dist`、`build`、`__pycache__`、`.next` 等明显不应提交的目录或文件出现在变更列表中。如果有，提醒用户确认是否需要先添加 `.gitignore`，而不是自动排除。
3. 运行 `git add -A` 暂存所有变更。
4. 根据变更内容生成一条中文约定式提交信息并执行 `git commit`。
5. 输出 commit hash 和提交信息。

## 提交信息格式

```
<type>: <中文描述>
```

- `scope` 默认省略，除非项目有明确模块划分且变更范围清晰。
- 描述简短具体，不超过 20 个字。

## 允许的类型

| 类型     | 含义                                   |
| -------- | -------------------------------------- |
| feat     | 新功能                                 |
| fix      | 问题修复                               |
| docs     | 仅文档变更                             |
| style    | 格式调整（空格、分号等），无逻辑变更   |
| refactor | 代码重构，既不是修复也不是新增功能     |
| perf     | 性能优化                               |
| test     | 添加或更新测试                         |
| chore    | 杂项                                   |
| build    | 构建系统变更（webpack、gulp、npm 等）  |
| ci       | CI/CD 配置变更（chart、Dockerfile 等） |
| revert   | 回滚之前的提交                         |

## 规则

- 默认提交所有变更，不推送。
- 不要提交密钥、令牌等敏感内容。
- 不要使用 `--no-verify`，除非用户明确要求。
- 不要 amend 已有提交，除非用户明确要求。
- 如果仓库已有提交信息规范，以仓库规范优先。
